Editor Workflow
Grid & Snap

Grid and Snap 워크플로 전면 개편

Scene View의 그리드에 커스텀 위치와 회전(rotation)을 부여할 수 있게 되었습니다. 기존에는 절대 그리드 스냅만 가능해서 직교 정렬 레벨에만 유용했지만, 이제 다리나 경사로처럼 회전된 레벨 요소에도 그리드를 맞출 수 있습니다.

주요 변경
Grid and Snap 오버레이 재설계, 단일 설정 메뉴 통합
새 기능
Grid Tool 핸들 회전, 커스텀 단축키, 컨텍스트 메뉴
워크플로 개선
Grid 크기와 Incremental Snap 크기 분리
접근
` 키 → Overlay 메뉴 → Grid and Snap
⚡ AEGIS 활용

5v5 전술 슈터 맵의 경사 지형, 비정렬 구조물(계단, 각진 통로) 배치 속도가 대폭 향상됩니다. 저사양 기기에서 정확한 콜리전 피팅을 위한 정밀 배치에 직접 도움됩니다.

Terrain

Terrain Layer 드래그 앤 드롭

Scene View와 Inspector 창에서 Terrain Layer를 드래그 앤 드롭으로 배치 가능합니다. Shader Graph에 Terrain 셰이더 템플릿이 추가되어 커스텀 지형 셰이더 작업도 간편해졌고, Terrain 노드가 각 Terrain과 TerrainData 에셋에 저장된 정보를 노출합니다.

드래그 앤 드롭
Scene View + Inspector에서 Terrain Layer 직접 배치
Shader Graph
Terrain 템플릿, Terrain 노드 (LOD 전환, max height 등)
⚡ AEGIS 활용

지형 텍스처링 작업 속도 향상. Shader Graph Terrain 템플릿으로 2GB RAM 디바이스 최적화된 커스텀 지형 셰이더를 더 쉽게 제작할 수 있습니다.

Core Engine
ECS

ECS — 코어 패키지로 통합

Entities, Collections, Mathematics, Entities Graphics 패키지가 Core 패키지로 편입되어 에디터와 함께 기본 배포됩니다. 기존 InstanceID는 새로운 EntityId 타입으로 대체되며, InstanceID 메서드는 deprecated 처리됩니다.

핵심 변경
ECS 4개 패키지 → Core package (에디터 내장)
ID 시스템
InstanceID → EntityId (단일 int 캐스팅 불가)
로드맵
ECS 컴포넌트를 GameObject에 직접 부착 (점진적 통합)
마이그레이션
EntityId ↔ int 캐스팅 코드 점검 필요
⚡ AEGIS 활용 — 높은 관련성

2GB RAM 타겟 디바이스 최적화의 핵심. M1 분석에서 발견된 Draw Call 병목(평균 727 vs 목표 200)을 해결하는 데 ECS 기반 렌더링 파이프라인이 직접적으로 도움됩니다. 코어 패키지 편입으로 프로젝트 셋업 복잡도도 감소. EntityId 전환에 따른 기존 코드 마이그레이션을 프로그래밍팀과 사전 점검 필요.

Performance

Microsoft DirectStorage 지원

NVMe 스토리지에서 텍스처, 메시, ECS/DOTS 데이터를 GPU로 직접 로드하여 로딩 시간을 최대 40%까지 단축합니다. Player Settings에서 "Enable Direct Storage" 옵션으로 활성화하며, Windows Standalone 빌드 대상입니다.

성능 향상
지원 에셋 타입 로드 시간 최대 40% 감소
플랫폼
Windows 10/11 (NVMe SSD 필요)
현재 제한
압축/해제에는 미적용 (C# API 개발 중)
활성화
Player Settings → Enable Direct Storage
⚡ AEGIS 활용

AEGIS는 모바일 타겟이므로 직접 적용 대상은 아닙니다. 다만 PC 개발/QA 빌드에서의 이터레이션 속도 향상, 향후 PC 크로스플랫폼 확장 시 중요한 기술 자산이 됩니다.

Analysis / QA
Analysis

Project Auditor — 에디터 기본 내장

이전에는 별도 패키지(com.unity.project-auditor)로 설치해야 했던 Project Auditor가 에디터에 기본 내장되었습니다. 프로젝트의 스크립트, 에셋, 코드를 정적 분석하여 성능 개선 포인트와 권장 사항을 한 번의 클릭으로 리포트합니다.

분석 항목
코드 할당, 셰이더 배리언트, SRP Batcher 호환성
접근
Window → Analysis → Project Auditor
출력
.projectauditor 파일 저장/로드, 어셈블리 필터링
셰이더 분석
패스 수, 키워드 수, 빌트 배리언트 수, GPU 인스턴싱
⚡ AEGIS 활용 — 높은 관련성

M2 빌드 퍼포먼스 분석 파이프라인에 즉시 통합 가능. 기존 Python 로그 분석으로 수동 점검하던 셰이더 배리언트, Draw Call 이슈를 에디터 안에서 자동 스캐닝. 저사양 기기 대상 셰이더 최적화 검증에 매우 유용합니다.

Statistics

Rendering Statistics 윈도우 리디자인

Rendering Statistics 윈도우가 시각적으로 전면 개편되었습니다. 이전 레거시 통계 창 대비 더 읽기 쉬운 레이아웃과 실시간 지표 확인이 가능합니다.

⚡ AEGIS 활용

Draw Call, 배칭 효율, 메모리 사용량 등을 실시간으로 확인하면서 씬 작업이 가능해져, 아트팀·프로그래밍팀 모두의 최적화 작업 효율이 올라갑니다.

Additional Notable Changes

Mesh LOD 디버깅

Scene View에서 LOD 레벨 라벨 토글, Inspector에서 카메라 비율별 LOD 레벨 확인 선택 바 추가

AEGIS: 캐릭터(프리드웬, 카야 등) LOD 셋팅 시각적 검증

Adaptive Performance

Basic 프로바이더가 PS4, PS5, Xbox Series X/S 콘솔 + 데스크톱 + 모바일 모두 지원

AEGIS: 동일 프레임워크로 다중 플랫폼 성능 스케일링 통일 관리

UI Toolkit

UXML/USS 파일 드래그 앤 드롭, 프로젝트 단위 UI 프리뷰 테마 설정, 버전 컨트롤 동기화

AEGIS: 3-Zone UI 구조 작업 시 UI Builder 워크플로 개선

Shader Graph

Terrain 셰이더 템플릿, "Delete and Reconnect" 노드 이름 변경, 단축키 개선

AEGIS: 저사양 기기용 커스텀 지형 셰이더 제작 간소화

Graph Toolkit 내장

Shader Graph, VFX Graph 기반인 Graph Toolkit이 에디터 빌트인 모듈로 편입

AEGIS: 패키지 설치 없이 즉시 그래프 도구 사용

XR / AR Foundation

마커 트래킹(QR코드), 메시 서브메시 분류, 레이캐스트 최대 거리 제약 API

AEGIS: 현재 해당 없음 (향후 AR 기능 확장 시 참고)

2D URP 개선

커스텀 렌더 패스를 C# 스크립트로 생성, 런타임 스프라이트 아틀라스 생성

AEGIS: 2D UI 오버레이 렌더링 최적화 가능성

Version Control

모든 뷰/다이얼로그 재디자인, 경로 축약 개선, 진행률 표시기, 오버레이 아이콘 개선

AEGIS: 팀 협업 시 VCS 사용성 향상
